## 三、关键业务变更分析

### 1. [重要] FAP 撤销功能上线 (fapSentRevocation)
**来源**: rrsjk-light-service (代继宁, commits b803392/d4ea220/68472b1) + rrsjk-admin-web (2bee0f9/55c34e6)
**证据等级**: 代码明确证明

**新增接口**:
- Admin: `POST /doRevocation.do` — 权限 `receipts:operate:sentRevocation`，批量撤销
- Service: `LightFapRecordService.doRevocation(orderNo, userName)`

**业务逻辑**:
1. 查询 FAP 记录，校验状态
2. 已记账 (SUCCESS) → 拒绝撤销
3. 已取消 (CANCEL) → 拒绝撤销
4. 已推送 (SENT) → 调用 `fapApi.cancelToNewEnergyFap()` 向新能源FAP系统发起作废
5. 更新 FAP 状态为 CANCEL，记录 cancelBy 和 fapCancelDesc
6. 清空对应业务订单的 fapStatus（回退到未推送状态）

**覆盖 11 种业务类型**: MATERIAL_REPURCHASE, STATION_REPURCHASE, SERVICE_DEPOSIT, SPARE_PARTS_DEPOSIT, INVERTER_SALE, MALL_ORDER, ZERO_CARBON_DEPOSIT, ZERO_CARBON_ORDER, VPP_ORDER, EPC_RECEIPT, ELECTRIC_ORDER

**业务意义**: FAP 已推送但未记账的单据现在可以被人工撤销，撤销后业务订单 fapStatus 清空，允许重新推送。

### 2. [重要] AISWEI 逆变器数据缓存大幅扩容
**来源**: rrsjk-light-data-service (yumiao, commit ace26201, 2026-06-12)
**证据等级**: 代码明确证明

**变更**:
- Guava Cache 容量: 15,000 → 250,000 (覆盖 ~12.4万 AISWEI SN + 预留)
- TTL: 5分钟 → 4小时
- 逆变器记录缓存: 8,000 → 250,000, TTL 2分钟 → 5分钟
- ThreadLocal 缓存增加上限: `AISWEI_THREAD_LOCAL_CACHE_MAX_SIZE = 4096`

**新增降级方法**:
- `fillEnergyDataFastFallback()` — 快速降级填充发电数据
- `setEnergyDataFromLatest()` — 从最新记录推算月/年发电量
- `setEnergyDataFallbackOnly()` — 纯降级（无缓存时用 eToday 替代）

**业务背景**: 生产 AISWEI 当前约 12.4 万个逆变器 SN，原缓存容量不足导致高峰期频繁 miss 打爆 DB。

### 3. Dubbo payload 增大至 16MB
**来源**: rrsjk-light-service, rrsjk-light-data-service, rrsjk-merchant-web (yumiao, 2026-06-12)
**证据等级**: 配置明确证明

**变更**: `service.xml` 中为 `lightStationService` 和 `lightInveterDataService` 添加 `<dubbo:parameter key="payload" value="16777216"/>`
**原因**: 逆变器数据量大，默认 Dubbo payload (8MB) 不够，需增大到 16MB。
