### 1. TAEI-2728: AI识别房产证 (测试中)

**负责人**: 杨越越 | **开发者**: 马斌(mabin) | **分支**: `origin/20260518-fangchanzhengOCR`

**业务语义**: 在并网确认和电站影像修改时，自动调用阿里云AI大模型识别房产证图片，提取业主姓名、身份证号、地址等信息，与电站原数据进行比对校验。

**完整业务链**:

```
前端上传房产证图片
    ↓
AcceptanceConfirmServiceImpl (rrsjk-light-impl)
    ├── gridConfirm() → houseCertificateOcr()  [line 747]
    └── modifyStationImg() → houseCertificateOcr()  [line 928]
    ↓
LightStationHouseCertificateOcrServiceImpl.houseCertificateOcr()
    ├── createData(): 创建OCR记录, 状态=running (AliStatusEnum)
    ├── alibabaApi.houseCertificateOcr(): 调用阿里云大模型API
    │   ├── 输入: stationName, fileUrl
    │   └── 输出: ResponseTextBody (code/data/message)
    └── 更新OCR结果:
        ├── success: ocrStatus=success, 解析name/id/add
        └── failed: ocrStatus=failed, 记录error message
    ↓
light_station_house_certificate_ocr 表 (MyBatis XML: 243行)
```

**新增实体**: `LightStationHouseCertificateOcr` (rrsjk-light-api)
| 字段 | 类型 | 含义 |
|------|------|------|
| id | Long | 主键 |
| stationCode | String | 电站编码 |
| houseCertificateUrl | String | 房产证URL |
| ocrStatus | String | OCR状态 (running/success/failed) |
| ocrResult | String | 大模型识别结果(JSON) |
| sourceStationName | String | 原业主姓名 |
| ocrStationName | String | OCR识别的业主姓名 |
| sourceIdNo | String | 原身份证号 |
| ocrIdNo | String | OCR识别的身份证号 |
| sourceAddress | String | 原地址 |
| ocrAddress | String | OCR识别的地址 |

**Dubbo服务**: `LightStationHouseCertificateOcrService` (service.xml line 808)

**关键逻辑** (代码明确证明):
- `checkIfExist()`: 按 stationCode + houseUrl + userName + idNo + address 去重，避免重复识别
- `selectLatest()`: 查询同一电站+图片的最新识别结果
- OCR结果解析: `result.getString("name")`, `result.getString("id")`, `result.getString("add").replace(" ","")`

**配套变更**:
- `admin-web`: 房产证AI识别页面、金蝶记账失败状态显示
- `alibabaApi`: 房产证OCR接口参数和URL配置修改
- 图片验真优化: 生产环境图片验证接口地址更新, 线程池配置优化
- 用户代理信息: `LightStationImageTemplate` 新增 userAgent 字段

**来源**: commit ee04e9fa73, cf7e5c4bf7, 323540e363 (2026-05-18~20)
