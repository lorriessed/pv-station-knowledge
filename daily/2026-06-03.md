## 关键业务变更

### 1. [重要] Admin 认证架构升级 — 从硬编码到 Dubbo authz-service

**影响仓库**: rrsjk-admin-auth-server, rrsjk-admin-authz-service

认证服务器 (`rrsjk-admin-auth-server`) 完成了关键的认证方式迁移：

- **移除硬编码**: `application.yml` 中的 `spring.security.user.name: admin / password: admin123` 已删除
- **新增 Dubbo 认证**: `AuthzUserDetailsService` 通过 Dubbo 调用 `rrsjk-admin-authz-service` 的 `AuthzUserService.getPasswordCredential()` 获取用户密码凭据
- **降级策略**: authz-service 不可用时 → 降级到 `bootstrap-admin` (环境变量 `BOOTSTRAP_ADMIN_ENABLED` 控制，默认关闭) → 最终 `UsernameNotFoundException`
- **密码编码**: Spring Security delegating password encoder (如 `{bcrypt}...`)
- **新表**: `authz_user_password` (user_id PK, password_hash, updated_by, updated_at)
- **新权限**: `authz:user:password` (重置用户密码) → 仅 `ADMIN_AUTHZ_SUPER` 角色

**生产影响**: 部署时需确保 authz-service 的 Dubbo 注册正常，否则 admin 用户将无法登录。

### 2. [重要] BFF 分中心管理迁移

**影响仓库**: rrsjk-admin-bff

旧版 CBS 分中心管理 (`cbs-web` → `SysSubCenterController`) 正在迁移到 BFF 架构：

- **新模块**: `subcentermanagement` 包下 3 个 Controller (SubCenter/Party/PartyUser) + 3 个 Dubbo Client
- **调用链**: admin-web-next (Vue3) → BFF REST → Dubbo → system-service
- **管理范围**: 分中心查询/用户管理、团体 CRUD、团体用户 CRUD

### 3. A 段政策按省份区分

- 彻底隐藏"并网规模奖励"功能
