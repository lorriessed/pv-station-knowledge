## 关键业务变更

### 1. [重要] rrsjk-admin-bff 管理后台 BFF 大规模功能扩展

**rrsjk-admin-bff** 从 dev 分支合并了大量新功能，是新一代管理后台的核心增长：

#### 1.1 操作日志审计体系 (全新)
- 基于 AOP (`@OperationLog`) 的管理后台操作日志记录
- 覆盖登录成功/失败/登出、角色CRUD、权限授权等所有写操作
- 通过 Dubbo 异步调用后端日志服务
- 新增 12 个类: `OperationLogAspect`, `LoginLogRecorder`, `DubboAdminOperationLogClient` 等
- 依赖新增: `spring-boot-starter-aop`, `rrsjk-admin-operation-log-api` (1.0.0-SNAPSHOT)

#### 1.2 数据大屏报表模块 (13个Controller)
- 能源中心收入日报、能源汇总日报、电站报表、电站成本报表
- 超期电站详情、资方交易报表、联合库存报表
- 分中心基础/建设/运营数据
- 统一 Client/Controller/Request/View 四层结构

#### 1.3 服务商与派单管理 (6个Controller)
- 施工队管理、派单管理、订单用户、合作方管理
- 服务商员工管理、子服务商管理
- 通过 Dubbo Client 调用 `rrsjk-light-service` 现有服务

#### 1.4 发电管理模块恢复
- 逆变器图表、逆变器数据、天气数据 Dubbo 客户端恢复
- InverterController / StationElectricityController 空值处理修复

#### 1.5 其他
- 租金管理: 共享项目付款账户导入恢复
- pom.xml 新增 `rrsjk-light-data-api` 依赖

### 2. 服务商保证金退款 operatorId 修复

- `LightDepositServiceImpl.refundDeposit()`: `operatorId` 从 `Long.valueOf(opUser)` 改为 `null`
- 修复原因: `opUser` 可能是服务商名称等非数字字符串，导致 NumberFormatException

### 3. 790W 模块类型支持

- `LightEnablePolicyItem` 新增 790W 模块类型
- 前端 FTL 模板 (add/edit) 同步更新

### 4. 逆变器表 Mapper XML 映射补齐

- `LightInveter.xml` columns 片段新增 `overvoltage_threshold_stage1`, `output_limit` 两列
- 此前 Java 实体已有字段但 local Mapper 缺少映射，本次补齐

### 5. 工商业电站写入工商业项目编码

- `CmLightProject.cmProjectCode` 在电站创建时写入
- `LightEpcStationModel` 增加赋值逻辑

### 6. 服务商日报查询优化

- `EnergySpDayServiceImpl`: `cityDao.findOneByCityCode()` 替代 `cityMap.get()` 直接查询
- 去除冗余日志输出
