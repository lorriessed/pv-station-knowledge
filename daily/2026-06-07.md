### ⭐ rrsjk-light-report-service: 报表 sub_center_name_new 回退为 sub_center_name (紧急修复)

**来源**: `rrsjk-light-report-service` → MyBatis Mapper XML (commits: f54f9bb/3ae2cbc/995aa0e/414ef7a/4bdaa3c, yumiao, 2026-06-07 12:03~15:48)
**证据等级**: 代码明确证明

**变更内容**: `LightStation.xml` 和 `LightStationPlanConfig.xml` 中将多个报表查询的 `sub_center_name_new` 字段全部回退为 `sub_center_name`：

| 查询方法 | 变更 |
|---|---|
| `findStationSum` | `a.sub_center_name_new as sub_center_name` → `a.sub_center_name` |
| `findStationSumUpgrade` | 3处: SELECT/GROUP BY/JOIN 条件全部从 `sub_center_name_new` 改为 `sub_center_name` |
| `findTotalStationByCityIdAndTimeAt` | `sub_center_name_new = #{subCenterName}` → `sub_center_name = #{subCenterName}` |
| `findExpiredThrCompleteStations` | 同上 |
| `findExpiredGridStations` | 同上 |
| `findStationPower` (LightStationPlanConfig.xml) | 2处: subquery alias 和 group by 回退 |
| `findStationGroupPower` (LightStationPlanConfig.xml) | 同上 |

**业务意义**: 这是对 `feature/202605/regionUnlimit` 分支上线后的**紧急修复**。TAEI-3053「一商做全国」上线后，新分中心数据直接写入 `sub_center_name`，`sub_center_name_new` 字段废弃（始终为 NULL）。regionUnlimit 分支上线时报表 mapper 误用了 `sub_center_name_new`，导致报表数据为空或错误，今日紧急回退。

**与已有知识一致**: `data/tables.md` 第 140 行已记录 "sub_center_name_new 废弃（始终为 NULL），报表必须用 sub_center_name"。

### rrsjk-admin-web: 分中心改造 — 3个 Controller 后续适配修复
