**问题回顾**: 2026-06-03 扫描发现 `LightFapRecordServiceImpl` 中经营性租赁FAP同步方法被注释，新Dubbo服务仅声明未使用。
**修复验证**: commit d5c920c634 (代继宁, 2026-06-05) 完整修复:
- `syncOperationalAssetFapSentStatus()` → `operationalAssetManagementService.udpateByPaymentReceiptCode()`
- `syncOperationalAssetFapResult()` → `operationalAssetManagementService.autoConfirmByPaymentReceiptCode(Map)`
- 新增 `BtFundAssetRepaymentRecordService` 基金FAP双向同步
- trade-service 商城订单记账状态/凭证/时间字段完善
- finance-service 收款记账凭证更新

**完整FAP路由**:
```
LightFapRecordServiceImpl (回调中心)
├── BizTypeEnum.OPERATIONAL_ASSET → OperationalAssetManagementService (rrsjk-light-report-service)
├── BizTypeEnum.LIGHT_FUND → BtFundAssetRepaymentRecordService (rrsjk-light-report-service)
├── BizTypeEnum.MALL_ORDER / ZERO_CARBON_ORDER → trade-service
├── BizTypeEnum.OPERATION_DEPOSIT / SPARE_PARTS_DEPOSIT → LightOperationDepositService (rrsjk-light-service)
└── BizTypeEnum.ZERO_CARBON_DEPOSIT → LightOperationDepositService (rrsjk-light-service)
```

### 3. 分中心权限重构
**架构变更**: admin-bff → admin-authz-service → system-service 分中心权限链路
- `AuthzSubCenterDTO` — 用户分中心范围（按 clientType 隔离）
- `CurrentUserSubCenterScope` — 分中心范围集中化管理
- 所有 BFF Controller 统一通过 `CurrentUserContext` 获取分中心范围
- **DDL**: `migration_20260604_authz_sub_center_scope.sql`
- **迁移中**: 旧 CBS 分中心管理 (cbs-web) → 新一代 BFF 架构 (rrsjk-admin-bff)

### 4. 电站方案变更暂停状态流转
**完整状态机**:
```
方案变更申请 → pauseStatus = PAUSED (冻结)
```
